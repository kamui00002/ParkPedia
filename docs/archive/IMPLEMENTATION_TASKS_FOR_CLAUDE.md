# 🤖 Claude Desktopで実装可能なタスク一覧

このファイルに記載されているタスクを、Claude Desktopにコピペして実装を依頼してください。

---

## ✅ 実装可能なタスク

### 1. 環境変数化の実装

**タスク内容**:
```
以下のタスクを実装してください：

1. .env.example ファイルを作成
   - Firebase設定のテンプレートを含める
   - 実際の値は含めず、プレースホルダーのみ

2. .gitignore に .env が含まれているか確認し、なければ追加

3. firebaseConfig.js が環境変数から読み込むようになっているか確認
   - 既に実装済みの場合は確認のみ
```

---

### 2. Firestoreルールの改善

**タスク内容**:
```
以下のタスクを実装してください：

1. firestore.rules ファイルを確認
2. reports, favorites, blockedUsers コレクションの list操作に厳格なクエリ制限を追加
3. クエリに userId フィルタを必須化
4. セキュリティを向上させる
```

---

### 3. console.logの整理

**タスク内容**:
```
以下のタスクを実装してください：

1. プロジェクト内の全ての console.log を検索
2. 本番環境でログが出力されないように、全ての console.log を __DEV__ で条件分岐
3. 例: console.log('...') → if (__DEV__) { console.log('...') }
4. console.error, console.warn も同様に処理
```

**注意**: 296箇所あるため、段階的に実装することを推奨します。

---

### 4. CI/CDパイプラインの構築

**タスク内容**:
```
以下のタスクを実装してください：

1. .github/workflows/ci.yml ファイルを作成
   - ESLint の実行
   - Prettier の実行
   - テストの実行
   - 型チェック（TypeScript）の実行

2. プッシュとプルリクエスト時に自動実行されるように設定
```

---

### 5. TypeScript型定義の追加（components/）

**タスク内容**:
```
以下のタスクを実装してください：

1. components/ ディレクトリ内のJavaScriptファイルを確認
2. 主要なコンポーネント（ErrorBoundary, CustomHeader, FilterDrawer, AdBanner など）の型定義を追加
3. 段階的にTypeScriptに移行するか、.d.ts ファイルで型定義を追加
```

## 📋 実装の優先順位

1. **最優先**: 環境変数化の実装（セキュリティ）
2. **高優先度**: Firestoreルールの改善（セキュリティ）
3. **中優先度**: CI/CDパイプラインの構築（品質保証）
4. **低優先度**: console.logの整理（段階的に実施）
5. **低優先度**: TypeScript型定義の追加（段階的に実施）
